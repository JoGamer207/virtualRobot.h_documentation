Dokumentation der Funktionen

◆ drawField()

void drawField ( )

Zeichnet das aktuelle Spielfeld Diese Funktion zeigt das Spielfeld mit dem Roboter und seinen Positionen an.

Zu beachten
Diese funktion wird nach jedem Move (Bewegung und Rotation) automatisch Ausgeführt. Es ist nicht nötig diese Funktion extra auszuführen,
kann aber dennoch ohne Probleme ausgeführt werden

◆ getNextPos()

int getNextPos ( char pos)

Gibt die Koordinaten des Feldes direkt vor dem Roboter zurück.

Parameter
posGibt an, ob X oder Y zurückgegeben werden soll
Rückgabe
Die Koordinaten vor dem Roboter
Veraltet
Diese funktion wird aktuell nicht mehr verwendet und ist eventuell nicht richtig Funktionsfähig
Warnung
Nicht richtig Funktionsfähig!

◆ getPos()

int getPos ( char pos)

Gibt die aktuelle Position des Roboters zurück.

Parameter
pos'x' für X-Koordinate, 'y' für Y-Koordinate
Rückgabe
Die angeforderte Koordinate

◆ getRotation()

int getRotation ( int opposite)

Bestimmt die Rotation des Roboters.

Parameter
oppositeGibt an, ob die Richtung hinter (1) oder vor (0) dem Roboter erfragt wird
Rückgabe
Die Richtung, in die der Roboter schaut oder hinter diesem, je nach Parameter

◆ initField()

int initField ( )

Initialisiert das Spielfeld und setzt die Hindernisse.

Rückgabe
0, wenn die Initialisierung erfolgreich war, sonst 1 bei Fehlern
Achtung
Muss am anfang der main funktion mindestens einmal ausgeführt werden, damit das Spielfeld funktioniert

Standard funktion - bearbeitet mit Feldüberprüfung und warnungssystem

◆ measureDistance()

int measureDistance ( )

Misst die Distanz zum nächsten Hindernis.

Rückgabe
Die Distanz zum nächsten Hindernis

Rückgabe = 1: Hindernis direkt vor dir - Rückgabe >=2: Distanz zum Hindernis - Rückgabe = 0: Kein Hindernis bis zum Maprand gefunden

◆ moveRobot()

void moveRobot ( int forward)

Bewegt den Roboter vorwärts oder rückwärts.

Parameter
forward1 für Vorwärtsbewegung, 0 für Rückwärtsbewegung

Standard funktion

◆ moveValid()

int moveValid ( )

Überprüft, ob der nächste Schritt im Spielfeld gültig ist.

Rückgabe
0 (Rand erreicht), 1 (Frei), 2 (Batterie erreicht), 3 (Bereits besucht)
Zu beachten
Rückgabewert 3 findet mit aktueller Version keine anwendung
